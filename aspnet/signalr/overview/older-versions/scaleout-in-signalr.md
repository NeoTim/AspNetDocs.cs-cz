<span data-ttu-id="f2105-146">Maximální propustnost, pomocí propojovací rozhraní, je nižší, než je, pokud klienti komunikovat přímo k uzlu jeden server.</span><span class="sxs-lookup"><span data-stu-id="f2105-146">Using a backplane, the maximum message throughput is lower than it is when clients talk directly to a single server node.</span></span> <span data-ttu-id="f2105-147">Důvodem je, propojovacího rozhraní předává všechny zprávy pro každý uzel, takže propojovacího rozhraní se může stát kritickým bodem.</span><span class="sxs-lookup"><span data-stu-id="f2105-147">That's because the backplane forwards every message to every node, so the backplane can become a bottleneck.</span></span> <span data-ttu-id="f2105-148">Záleží na aplikaci, zda toto omezení je nějaký problém.</span><span class="sxs-lookup"><span data-stu-id="f2105-148">Whether this limitation is a problem depends on the application.</span></span> <span data-ttu-id="f2105-149">Například tady jsou některé typické scénáře SignalR:</span><span class="sxs-lookup"><span data-stu-id="f2105-149">For example, here are some typical SignalR scenarios:</span></span>
