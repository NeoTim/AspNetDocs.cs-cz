<span data-ttu-id="b964b-107">RESTful služeb, jako je ASP.NET Web API zjednodušit vytváření aplikací založených na protokolu HTTP pro vývojáře tím, že poskytuje abstrakci Architektura aplikace na straně serveru a na straně klienta.</span><span class="sxs-lookup"><span data-stu-id="b964b-107">RESTful services like ASP.NET Web API simplify the creation of HTTP-based applications for developers by abstracting the architecture for server-side and client-side applications.</span></span> <span data-ttu-id="b964b-108">Místo vytváření vlastnickým protokolem založené na soket pro komunikaci, webové rozhraní API vývojářům jednoduše je potřeba publikovat metody požadavku HTTP pro svou aplikaci (například: GET, POST, PUT, DELETE), a vývojáři klientských aplikací můžou potřebují jenom využívat metody HTTP, které jsou nezbytné pro svou aplikaci.</span><span class="sxs-lookup"><span data-stu-id="b964b-108">Instead of creating a proprietary socket-based protocol for communication, Web API developers simply need to publish the requisite HTTP methods for their application, (for example: GET, POST, PUT, DELETE), and client application developers only need to consume the HTTP methods that are necessary for their application.</span></span>
